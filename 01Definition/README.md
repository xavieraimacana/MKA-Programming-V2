# Definición del Proyecto: Sistema de Gestión de Cafetería

Este documento resume la definición del problema y la descripción general del proyecto según la documentación oficial.

## 1. Descripción del Problema
Una cafetería mediana/pequeña enfrenta ineficiencias operativas debido al registro y control manual de sus procesos internos. Los principales problemas identificados son:
*   **Descontrol de Inventario:** Pérdidas por desperdicio de insumos perecederos y desabastecimiento de ingredientes clave (café, leche, pan, etc.) durante horas pico.
*   **Seguimiento Ineficiente de Ventas:** Registro de transacciones manual o informal que genera errores contables y dificulta el balance financiero diario.
*   **Gestión de Pedidos Deficiente:** Retrasos y entregas incorrectas en horas de alta afluencia por falta de una cola ordenada.
*   **Falta de Trazabilidad e Informes:** Ausencia de registros estructurados que permitan evaluar el rendimiento del negocio y tomar decisiones oportunas.

## 2. Objetivo del Sistema
Desarrollar una aplicación de consola en Java (Fase 1) estructurada bajo el patrón **Model-View-Controller (MVC)** y persistida mediante **MongoDB**, que permita automatizar el punto de venta (POS), la pantalla de cocina (KDS), el control de inventarios basado en recetas y la generación de reportes diarios (Reporte Z) de manera segura y eficiente.
